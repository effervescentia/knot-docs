# Elements

This expression is used to describe an element in the application's render tree.
The syntax is very similar to [React JSX](https://reactjs.org/docs/introducing-jsx.html).

Most [views](declaration/view) will return an element to be rendered.

## Definition

Element primitives can be constructed outside of views though they will not be connected to render tree
until they are used within a rendered view. If an element has no children then it can be written as self-closing.

<Tabs>
<TabItem value="knot" label="Knot">

```knot
const url = "https://example.com/image.png";

const element = <img src=url></img>;
const selfClosing = <img src=url />;
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx
const url = "https://example.com/image.png";

const element = <img src={url}></img>;
const selfClosing = <img src={url} />;
```

</TabItem>
</Tabs>

## Adding Parentheses

Most property expressions do not require explicitly adding parentheses, but for complex expressions
it can help to better indicate the end of an expression.

<Tabs>
<TabItem value="knot" label="Knot">

```knot
const stringAttr = <input value="foo bar" />;
const booleanAttr = <input disabled=true />;
const numericAttr = <input tabindex=(123 + 456 + 789) />;
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx
const stringAttr = <input value="foo bar" />;
const booleanAttr = <input disabled={true} />;
const numericAttr = <input tabindex={123 + 456 + 789} />;
```

</TabItem>
</Tabs>

## Attribute Punning

You can easily provide the attributes of an element from a variable with a matching name in the same scope.

<Tabs>
<TabItem value="knot" label="Knot">

```knot
const name = "foo";
const counter = 10;

const element = <Widget name counter />;
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx
const name = "foo";
const counter = 10;

const element = <Widget name={name} counter={counter} />;
```

</TabItem>
</Tabs>

:::caution
Note that this means unlike JSX an attribute without a value does not have a value of `true`.

<Tabs>
<TabItem value="knot" label="Knot">

```knot
const element = <input disabled=true />;
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx
const element = <input disabled />;
```

</TabItem>
</Tabs>
:::

## Inline Expressions

Expressions can be added inline by wrapping it in braces (`{ }`) to provide the inner content of an element.

<Tabs>
<TabItem value="knot" label="Knot">

```knot
func hello -> "Hello, world!";

const inline = <h1>{hello()}</h1>;
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx
function hello() {
  return "Hello, world!";
}

const inline = <h1>{hello()}</h1>;
```

</TabItem>
</Tabs>

## Text Content

Simple textual content can be written inline to provide the inner content of an element.

<Tabs>
<TabItem value="knot" label="Knot">

```knot
const inline = <h1>Hello, world!</h1>;
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx
const inline = <h1>Hello, world!</h1>;
```

</TabItem>
</Tabs>
