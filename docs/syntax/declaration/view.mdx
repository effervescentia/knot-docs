# Views

An entity that accepts a collection of properties and returns an [element](../element).

## Definition

Although similar in syntax to a [function](function), the property order for views doesn't matter as
it always produces the same type signature. The only valid return types for a view are the [primitives](../primitive).
You can render a view the same way you might render a built-in element such as `<div>` in a browser environment.

<Tabs>
<TabItem value="knot" label="Knot">

```knot
view SimpleView(age: integer, name: string) -> <div />;

view AlternateView(name: string, age: integer) -> <div />;

const element =
  <>
    <SimpleView age=4 name="first" />
    <SimpleView name="second" age=3 />
    <AlternateView age=2 name="third" />
    <AlternateView name="fourth" age=1 />
  </>;
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx
function SimpleView({ age, name }: { age: number; name: string }) {
  return <div />;
}

function AlternateView({ age, name }: { name: string; age: number }) {
  return <div />;
}

const element = (
  <>
    <SimpleView age={4} name="first" />
    <SimpleView name="second" age={3} />
    <AlternateView age={2} name="third" />
    <AlternateView name="fourth" age={1} />
  </>
);
```

</TabItem>
</Tabs>

## Empty Properties

If your view does not have any properties you can drop the parentheses as well.

<Tabs>
<TabItem value="knot" label="Knot">

```knot
view SimpleView -> <div />;

const element = <SimpleView />;
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx
function SimpleView() {
  return <div />;
}

const element = <SimpleView />;
```

</TabItem>
</Tabs>

## Default Properties

You can make your properties optional by passing default values.
Unlike a function, there are no restrictions about which properties can have default values.

<Tabs>
<TabItem value="knot" label="Knot">

```knot
view PartialView(first: integer, second = 10, third: integer) ->
  <div>{first + second + third}</div>;

const element = <PartialView first=1 third=3 />;
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx
function PartialView({
  first,
  second = 10,
  third,
}: {
  first: number;
  second?: number;
  third: number;
}) {
  return <div>{first + second + third}</div>;
}

const element = <PartialView first={1} third={3} />;
```

</TabItem>
</Tabs>

## Styling

To control the rendering of the elements in a view you can apply styles either in an element
expression or by creating a "styled" element which can itself be used in element expressions.

### Styled Elements

The recommended way to use styles is to bake them directly into elements which can then be
easily reused between views.

<Tabs>
<TabItem value="knot" label="Knot">

```knot
const headerStyles = style {
  fontSize: $px(10),
};

const Header = div::headerStyles;
const Content = div::{
  fontSize: $px(8),
};

const element =
  <div>
    <Header>Welcome!</Header>
    <Content>This is my blog.</Content>
  </div>;
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx
// a similar API is also provided by @emotion/styled, styled-components, styled-jss, etc.
import { styled } from '@stitches/react';

const headerStyles = {
  fontSize: '10px',
};

const Header = styled('div', headerStyles);
const Content = styled('div', {
  fontSize: '8px',
});

const element = (
  <div>
    <Header>Welcome!</Header>
    <Content>This is my blog.</Content>
  </div>
);
```

</TabItem>
</Tabs>

### Styling Element Expressions

Styles can also be applied directly to elements within an expression.

<Tabs>
<TabItem value="knot" label="Knot">

```knot
const header = style {
  fontSize: $px(10),
};

const element =
  <div>
    <div::header>Welcome!</div>
    <div::{ fontSize: $px(8) }>This is my blog.</div>
  </div>;
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx
const header = {
  fontSize: '10px',
};

const element = (
  <div>
    <div style={header}>Welcome!</div>
    <div style={{ fontSize: '8px' }}>This is my blog.</div>
  </div>
);
```

</TabItem>
</Tabs>
