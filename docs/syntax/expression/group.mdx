# Groups

If you need to overcome the [operator](../operator) priority you can use groups to represent your intended logical groupings.

<Tabs>
<TabItem value="knot" label="Knot">

```knot
const falseResult = 123 - 4 * 10 > 90;
const trueResult = (123 - 4) * 10 > 90;
```

</TabItem>
<TabItem value="tsx" label="TypeScript">

```tsx
const falseResult = 123 - 4 * 10 > 90;
const trueResult = (123 - 4) * 10 > 90;
```

</TabItem>
</Tabs>
